Direkte Sortierverfahren

Selectionsort

Das Array wird sequentiell nach dem kleinsten Element gesucht. Wird ein kleineres Element gefunden wird dieses an die erste Stelle gesetzt. Ist man das Array einmal durchlaufen sucht man nach dem kleinsten Element aus dem restlichen Array und stellt dieses an die zweite Stelle. ...

Insertionsort

Beim Insertionsort betrachtet man von links nach rechts jedes Element im Array. Bei jedem neuen Element vergleicht man es mit dem linken Teil des Arrays und fügt es an der passenden Stelle ein. Alle darauffolgenden Zahlen des linken Teils müssen dann aufrücken.

Bubblesort

Beim Subblesort beginnt man vorne im Array und vergleicht immer ein Element mit seinem direkten Nachbarn. Ist das linke Element größer als das rechte dann tauschen die beiden Position. So erreicht man das man nach einem Array durchlauf das größte Element ganz rechts hat.

Bewertung von Algorithmen

Für einen Algorithmus ist es neben der Korrektheit wichtig, dass er einen möglichst geringen Ressorucenverbrauch hat. Der Ressourcenverbrauch ist ein Zusammenspiel aus Zeitbedarf und Speicherplatzbedarf /n Zeit- und Speicherbedarf hängen dabei stark von der zu verarbeitenden Datenmenge ab. Es gilt prinzipiell das eine größere Eingabemenge auch die Laufzeit verlängert. Die Frage zur Beurteilung eines Algorithmus lautet also: Um wieviel wächst die Laufzeit oder der Speicherplatzbedarf, wenn sich die Eingabemenge verdoppelt?
